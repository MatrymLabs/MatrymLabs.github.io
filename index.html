Case study

Debugging like a field engineer — measured, not guessed

Problem. While walking the whole engine end-to-end over a live socket, every command felt sluggish — a flat ~40 ms, whether it did real work or not.

Investigation. A uniform delay across cheap and expensive commands isn't compute — it's the network. The gateway never set TCP_NODELAY, so each one-line reply stalled ~40 ms on Nagle + delayed-ACK.

Fix & proof. One setsockopt. Re-measured with the same end-to-end test: ~40 ms → ~1 ms per command — a ~20–40× latency drop for every client. Same method later cut the test suite 2.3× by finding the one thing that was actually slow.

The point isn't the fix — it's the method: measure, find the one thing that's slow, fix it, re-measure. No shotgun optimization.
