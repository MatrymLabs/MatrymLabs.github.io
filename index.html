The proof-tool

forge-audit - proof, not assertion

A portfolio makes a claim: this engineer uses AI as a force multiplier, wrapped in automated evidence and quality gates. Rather than assert it, I built the tool that proves it.

Point it at a repo; it grades the repo

forge-audit runs the quality gates (ruff · mypy · pytest --cov · bandit · pip-audit) on any repository - using that repo's own toolchain and declared scope - and emits a machine-checkable JSON scorecard (pass · watchlist · fail) graded against objective stage thresholds. The network and the toolchain sit behind mockable seams, so its own tests never touch either.

It even grades a repo it did not build honestly: when it lacks that repo's dev environment it marks the affected gate not graded here, never a failure - it will not call a green-CI project broken just because a dependency is missing. The rule it holds itself to is the rule it grades by: no claim without correspondence.

Its reading of CodeForge: lint · typecheck · tests · security · dependencies · CI · collaboration · performance all pass - a clean scorecard, graded by the same rule it grades anything else: no claim without correspondence. Coverage sits above the advanced bar; the live numbers are the CI badge, not a figure typed here (which would only drift).

Fleet mode points it at every repo on the ship in one run and rolls the per-repo verdicts into a single fleet verdict - platform-style tooling, not a one-repo script. CodeForge (public) is graded above; the private repos are audited by the same tool and shared on request.

Case study

Debugging like a field engineer - measured, not guessed

Problem. While walking the whole engine end-to-end over a live socket, every command felt sluggish - a flat ~40 ms, whether it did real work or not.

Investigation. A uniform delay across cheap and expensive commands isn't compute - it's the network. The gateway never set TCP_NODELAY, so each one-line reply stalled ~40 ms on Nagle + delayed-ACK.

Fix & proof. One setsockopt. Re-measured with the same end-to-end test: ~40 ms → ~1 ms per command - a ~20-40× latency drop for every client. Same method later cut the test suite 2.3× by finding the one thing that was actually slow.

The point isn't the fix - it's the method: measure, find the one thing that's slow, fix it, re-measure. No shotgun optimization.

Full-stack

CodeForge Console - the front/back split, typed end to end

The decoupled architecture employers ask for: a React front end talking to a Python API over a typed contract.

A separate front end, not a server-rendered page

codeforge-console is a React + TypeScript + Next.js single-page app that consumes CodeForge's typed /api/status contract and renders its live evidence board (career · QA · hardware · performance). The TypeScript types mirror the FastAPI/Pydantic models, so the front/back seam is typed on both sides; one fetch layer is the only door to the API, and an honest banner shows when the API is down.

Tests run on Vitest + React Testing Library with the network mocked - the same seam discipline as the Python repos - and CI runs lint · typecheck · test · build green on every merge.

Performance engineering

pyg-perf-lab - GNN performance, measured honestly

Profilers and benchmarks for graph-neural-network training, built to one rule: nothing is claimed as measured that was not actually run.

Measured, and honest about what was measured

pyg-perf-lab is a PyTorch Geometric performance-engineering package: runnable profilers and benchmarks for GNN training, PyCharm run-configs, Nsight wrappers, a layered tuning guide, and CI. Where a lot of ML portfolios quietly claim GPU numbers they never ran, this one draws the line in the README: the CPU paths (config/timing logic, the full_batch_forward benchmark, CPU torch.profiler) are run and verified on the author's GPU-less primary host; the GPU benchmark and profiler paths are verified on a separate RTX 4090 (a dated snapshot, not a live claim); and the Nsight wrappers stay labelled authored-not-run. Every line says which host measured what.

Same thesis as everything else on this page: no claim without correspondence. The honesty is the feature.
